# 用户登录功能测试用例文档

**版本**: 1.0  |  **创建日期**: 2026-03-06

## 1. 测试用例概述

- **项目名称**: 用户登录系统
- **功能模块**: 用户登录功能
- **测试类型**: 功能测试
- **测试级别**: 系统测试
- **测试环境**: 测试环境
- **测试工具**: Postman, Selenium, JMeter
- **测试人员**: 测试工程师

## 2. 测试目标

- 验证用户能够正常登录系统
- 验证登录失败时的错误处理机制
- 验证登录安全性
- 验证登录性能
- 验证用户体验

## 3. 测试范围

- 用户名/密码登录
- 记住密码功能
- 忘记密码功能
- 验证码功能
- 第三方登录（如微信、QQ）
- 登录日志记录
- 会话管理
- 安全防护（防暴力破解）

## 4. 详细测试用例

### TC-LOGIN-001 - 正常登录测试

**优先级**: 高  |  **测试类型**: 正向测试

**前置条件**:
1. 用户已注册
2. 用户账号状态正常

**测试步骤**:
1. 打开登录页面
2. 输入正确的用户名
3. 输入正确的密码
4. 点击登录按钮

**预期结果**:
1. 登录成功
2. 跳转到首页或指定页面
3. 显示登录成功提示
4. 记录登录日志

---

### TC-LOGIN-004 - 记住密码功能测试

### TC-LOGIN-005 - 忘记密码功能测试

**优先级**: 高  |  **测试类型**: 功能测试

**前置条件**:
1. 用户已注册

**测试步骤**:
1. 打开登录页面
2. 点击'忘记密码'链接
3. 输入注册邮箱/手机号
4. 点击发送验证码
5. 输入收到的验证码
6. 设置新密码
7. 使用新密码登录

**预期结果**:
1. 成功发送验证码
2. 验证码验证通过
3. 密码重置成功
4. 可以使用新密码登录

### TC-LOGIN-007 - 连续登录失败锁定测试

**优先级**: 高  |  **测试类型**: 安全测试

**前置条件**:
1. 用户已注册

**测试步骤**:
1. 打开登录页面
2. 连续输入错误密码5次
3. 第6次输入正确密码尝试登录

**预期结果**:
1. 前5次显示错误提示
2. 第6次显示：'账号已锁定，请30分钟后重试'
3. 账号被临时锁定

---

### TC-LOGIN-008 - 会话超时测试

**优先级**: 中  |  **测试类型**: 功能测试

**前置条件**:
1. 用户已登录

**测试步骤**:
1. 登录系统
2. 保持页面不操作30分钟
3. 尝试操作页面

**预期结果**:
1. 会话超时
2. 自动跳转到登录页面
3. 显示会话超时提示

---

### TC-LOGIN-009 - 多设备登录测试

**优先级**: 中  |  **测试类型**: 功能测试

**前置条件**:
1. 用户已注册

**测试步骤**:
1. 在设备A上登录
2. 在设备B上使用相同账号登录
3. 检查设备A的登录状态

**预期结果**:
1. 设备B登录成功
2. 设备A被强制退出
3. 设备A显示：'账号在其他设备登录'

---

### TC-LOGIN-010 - 性能测试

**优先级**: 中  |  **测试类型**: 性能测试

**前置条件**:
1. 测试环境准备就绪

**测试步骤**:
1. 使用JMeter模拟100个并发用户登录
2. 监控服务器响应时间
3. 检查系统资源使用情况

**预期结果**:
1. 平均响应时间 < 2秒
2. 成功率 > 99%
3. CPU使用率 < 80%
4. 内存使用率 < 80%

## 6. 风险评估

| 风险项 | 风险等级 | 应对措施 |
|--------|----------|----------|
| SQL注入攻击 | 高 | 使用参数化查询，输入验证 |
| 暴力破解攻击 | 高 | 登录失败锁定机制 |
| 会话劫持 | 中 | 使用HTTPS，设置安全Cookie |
| 密码泄露 | 高 | 密码加密存储，定期更换 |
